# Сортировка с ограничением

Вам дан целочисленный массив `nums` и три параметра:
- `maxValue`: максимальное значение элемента в массиве.
- `maxCount`: максимальное количество операций сравнения, которое вы можете выполнить.
- `maxMemory`: максимальный объем дополнительной памяти (в ячейках), который вы можете использовать.

Ваша задача — отсортировать массив в порядке неубывания, соблюдая **все** ограничения.

**Ограничения:**
- Длина массива: `1 <= nums.length <= 10^5`
- `-10^5 <= nums[i] <= maxValue <= 10^5`
- `1 <= maxCount <= 10^12`
- `1 <= maxMemory <= 10^6`

**Определения:**
- **Операция сравнения** — это любая операция, в которой два элемента массива сравниваются друг с другом (например, `a < b` или `nums[i] > nums[j]`).
- **Дополнительная память** — это любое пространство, используемое сверх исходного массива. Считается в "ячейках" (например, одна переменная типа `int` — это одна ячейка, создание дополнительного массива длины `k` — это `k` ячеек). Память под временные переменные в вашем алгоритме также должна учитываться.

**Вам необходимо реализовать функцию:**
function sortWithConstraints(nums, maxValue, maxCount, maxMemory)

**Важно:** Вы должны выбрать алгоритм сортировки, который гарантированно выполнит сортировку, не превысив ни одно из ограничений. Использование встроенных функций сортировки запрещено.

---

**Пример 1:**

Вход: nums = [3, 1, 2], maxValue = 100, maxCount = 1000, maxMemory = 1000
Выход: [1, 2, 3]
Объяснение: Ограничения очень мягкие, подойдет любой алгоритм (например, быстрая сортировка или сортировка слиянием).

**Пример 2:**

Вход: nums = [100000, -100000], maxValue = 100000, maxCount = 1, maxMemory = 1000000
Выход: [-100000, 100000]
Объяснение: Ограничение на сравнения очень строгое, но массив маленький, а память большая. Идеально подходит сортировка подсчетом, которая требует O(n + k) памяти и O(n + k) операций, но минимальное количество сравнений.

**Пример 3:**

Вход: nums = [5, 4, 3, 2, 1], maxValue = 5, maxCount = 30, maxMemory = 5
Выход: [1, 2, 3, 4, 5]
Объяснение: Ограничение по памяти очень строгое (нельзя создавать большой дополнительный массив). Однако ограничение на сравнения довольно свободное. Здесь подойдет сортировка вставками или выбором, которые используют O(1) дополнительной памяти.
